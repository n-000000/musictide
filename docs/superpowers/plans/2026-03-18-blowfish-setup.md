# Blowfish Hugo Site Setup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold a clean Hugo site at `/home/n0xx/Code/infra/service/musictide` with Blowfish installed as a Hugo module, bilingual (pt-PT primary, en-GB secondary), verified working with `yarn watch`.

**Architecture:** Hugo generates static HTML from Markdown content using Blowfish templates. Blowfish is pulled as a Go module — no vendored theme directory. Config is split across `config/_default/` YAML files following Hugo's split-config convention.

**Tech Stack:** Hugo Extended ≥ 0.141.0 (0.158.0 installed), Blowfish v2 (`github.com/nunocoracao/blowfish/v2`), YAML config throughout, Yarn for script runner.

## Task 6: Markup Config

**Files:**
- Create: `config/_default/markup.yaml`

Blowfish's docs state this file is required for the theme to function correctly. It configures Goldmark (Hugo's markdown renderer) and syntax highlighting.

- [ ] **Step 6.1: Create `config/_default/markup.yaml`**

```yaml
goldmark:
  parser:
    wrapStandAloneImageWithinParagraph: false
    attribute:
      block: true
  renderer:
    unsafe: true
  extensions:
    passthrough:
      enable: true
      delimiters:
        block:
          - ['\[', '\]']
          - ['$$', '$$']
        inline:
          - ['\(', '\)']

highlight:
  noClasses: false

tableOfContents:
  startLevel: 2
  endLevel: 4
```

`goldmark.renderer.unsafe: true` is required to allow raw HTML in Markdown — necessary for rich editorial articles with inline video embeds. `highlight.noClasses: false` tells Hugo to use CSS classes for syntax highlighting rather than inline styles, which Blowfish's stylesheet expects.

- [ ] **Step 6.2: Commit**

```bash
git add config/_default/markup.yaml
git commit -m "chore: markup config (goldmark + highlight for blowfish)"
```
